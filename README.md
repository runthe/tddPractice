#TDD
+ 제품 코드 만들기 전에 테스트 먼저 작성하기
+ 기본 흐름 (완료할 때 까지 반복)
+ 실패 테스트 작성 -> 테스트 성공 시키기 -> 코드 청소하기
+ 테스트 작성 순서
  - 쉬운것, 예외적인 것 -> 어려운 것, 정상적인 것


#AuthService
+ 테스트 클래스 만들기
+ 객체 생성하기(쉬운)
+ ID 값이 비정상적인 경우(쉬운 정상에서 벗어난)
+ PW 값이 비정상적인 경우(쉬운 정상에서 벗어난)
+ User가 존재하지 않는경우(정상에서 벗어난)
+ ID에 해당하지 않는 User가 존재하는데, PW가 일치하지 않는 경우(정상에서 벗어난)
+ ID와 PW가 일치하는 경우(정상)
  - 인증 정보를 리턴
